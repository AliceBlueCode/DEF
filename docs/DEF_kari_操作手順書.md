# DEF(kari) 操作手順書 v1.0.0

## 1. 起動

```bash
cd DEF
streamlit run def_kari/app.py
```

ブラウザで `http://localhost:8510` が開きます。

---

## 2. 初回セットアップ

1. `.env.example` を `.env` にコピー
2. `.env` にバックエンドのディレクトリパスを設定（使用するもののみ）
3. 設定タブの「🔑 APIキー管理」から外部APIキーを設定（必要な場合のみ）
4. 設定タブ下部の「💾 保存」を押す

---

## 3. タブ構成

| タブ | 用途 |
|---|---|
| 👤 キャラクター | キャラクターのプロフィール閲覧・編集 |
| 💬 チャット | 1対1の対話 |
| 🎭 セッション | 複数AI＋人間の卓 |
| 📖 エピソード | 小説執筆＋AI候補生成 |
| 🤔 思考 | フリーテキストでのAI思考実験 |
| ⚙ 設定 | バックエンド・レーティング・各種設定 |
| 🐛 デバッグ | LLM生応答・フォールバック確認 |

---

## 4. チャットタブ

### 基本操作
1. サイドバーでキャラクターを選択
2. テキスト入力欄にメッセージを入力して送信
3. AIが応答（テキスト＋音声＋画像）

### 主なボタン
- **🔄 再生成** — 最後の応答を再生成
- **↩ 元に戻す / ↪ やり直す** — Undo/Redo
- **🗑 削除** — 最後の応答を削除

---

## 5. セッションタブ

### セッション開始
1. 参加キャラクターを選択（複数可）
2. セッションルールを選択
3. 「▶ 開始」ボタンでセッション開始

### セッション中の操作
- **自動進行チェックボックス** — ONでAIが自動で会話を進行
- **発言完了** — AIの発言を完了させて次のターンへ
- **割り込み** — 発言力-2で割り込み発言
- **次発言者指名** — 発言力-1で次の発言者を指定
- **投票** — お題変更・退場・終了の投票を発議
- **🎨 挿図** — 場面のイラストを生成

### 人間プレイヤー
- `player_type: "human"`のキャラクターを参加させると人間入力モードになる
- 自分のターンにテキスト入力欄が表示される

---

## 6. エピソードタブ

### 基本操作
1. 「+ 新規作品」を選択してタイトルを入力
2. 本文を書く
3. 「✍ 生成」でAI候補を生成（設定した候補数分）
4. 右カラムのタブで候補を比較
5. 「⬇ 本文に追加」で気に入った候補を本文に追加
6. 「💾 保存」で保存

### プロット設定
- 「📝 プロット設定」ボタンでダイアログを開く
- 📂でプロットファイルを読み込み、または直接編集
- 「💾 保存」でエピソードJSONに保存
- 「✅ 反映」で保存＋AIのシステムプロンプトとして適用

### Chapter/Scene
- **新章** — `--- Chapter N ---` + `--- Scene 1 ---` を本文に挿入
- **新場面** — `--- Scene N ---` を本文に挿入

### TTS読み上げ・T2I挿絵
- **🔊 現場面** — 最後のSceneを読み上げ
- **🔊 選択読上** — プルダウンで選んだSceneを読み上げ
- **🎨 現挿絵** — 最後のSceneの挿絵を生成
- **🎨 選択挿絵** — プルダウンで選んだSceneの挿絵を生成
- **⚙ T2I** — T2Iバックエンド・モデルの切替ダイアログ

### LLMバックエンド切替
- プロット設定ボタンの右のプルダウンでエピソード専用のLLMを選択可能

---

## 7. 設定タブ

### 構成
- **母国語** — UI表示言語の切替（日英中韓西仏独）
- **LLMバックエンド** — TGW / OpenAI / Gemini / Anthropic
- **TTSバックエンド** — VOICEVOX / Kokoro / Irodori / Gemini TTS
- **T2Iバックエンド** — A1111 / ComfyUI / HuggingFace / Civitai
- **C2方式** — 画像プロンプト翻訳プロバイダ
- **チャットモード** — 挨拶ON/OFF、Undo保持件数
- **セッションモード** — アクション数/ターン、繰り返しペナルティ、挿図サイズ
- **エピソードモード** — 生成候補数、挿絵サイズ
- **APIキー管理** — 外部APIキーの暗号化保存

### 保存
設定変更後は必ず「💾 保存」ボタンを押してください。F5（ブラウザリロード）で保存した設定が反映されます。

---

## 8. サイドバー

- **キャラクター選択** — チャット対象の切替
- **レーティング設定** — 性的/暴力の許容レベル（各4段階）
- **セーフティレベル** — off / warn / mask
- **TTS設定** — 音声生成ON/OFF、人間プレイヤー音声ON/OFF
- **T2Iトリガー** — 各サイクル最後 / 各サイクル最初 / 手動 / インターバル
- **D方式** — 感情補助タグON/OFF

## 9. データの配置

```
data/
├── public/              # 公開データ（git管理対象）
│   ├── characters/      # 公開キャラクター
│   ├── action_directives/
│   ├── session_rules/
│   └── episode_prompts/
├── private/             # プライベートデータ（git除外）
│   ├── characters/      # NSFWキャラクター
│   ├── episodes/        # エピソード作品
│   ├── episode_prompts/ # プライベートプロット
│   ├── session_history/
│   ├── session_rules/
│   ├── action_directives/
│   └── thoughts/
└── llm_profiles/        # LLMモデルプロファイル
```

---

## 10. キャラクターの追加

1. `data/public/characters/`（または`data/private/characters/`）に新しいディレクトリを作成
2. ディレクトリ名がキャラクターIDになる（例: `character_mychar_001`）
3. `profile.json`を作成（既存キャラクターをテンプレートとして利用可）
4. 必要に応じて`icon.png`（512×512）と`standing.png`（832×1216）を配置
5. アプリをリロードするとプルダウンに表示される

---

## 11. トラブルシューティング

| 症状 | 対処 |
|---|---|
| バックエンドが起動しない | `.env`のディレクトリパスを確認。空欄のバックエンドは自動起動しない |
| 設定がF5で戻る | 設定タブの「💾 保存」を押してから再読み込み |
| 音声が再生されない | サイドバーのTTS設定がONになっているか確認 |
| 画像が生成されない | T2Iトリガーが「手動」以外になっているか確認 |
| LLMの応答が日本語でない | 設定タブの母国語設定を確認 |
| コンテキスト上限エラー | セッション履歴が長すぎる。新しいセッションを開始 |
